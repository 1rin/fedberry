FedBerry is a Fedora Remix specifically built for use with the Raspberry PiÂ® 2 Model B computer.

FedBerry was started by Vaughan Agrez in Oct 2015, then in Jan 2016 Gor Martsen joined the project.

As Fedora does not (yet) support the Raspberry Pi 2 Model B (RPi2), FedBerry was created so people could easily install and use Fedora on the RPi2.

FedBerry is still in its infancy with two 2 active developers working on the project. If you wish to contribute, please feel free to join us.

The FedBerry team would like to say a big thank you to all the Fedora developers that make this remix possible. We appeciate your dedication and hard work and are greatful that you make it freely available to all.

  • Mini release

    This release differs significantly from default Fedora settings used in their arm releases. It is a small 'bare bones' remix with some optimizations specifically tailored towards RPi2 usage. Its main goals are to have a small footprint, fast boot times and to reduce the number of writes to the SD card (within reason).

    The root partition is NOT automatically re-sized at boot due to boot time and image size considerations. The initial-setup utility is not used for this release.

  • Minimal and XFCE release

    Both these releases are close to 'stock' arm releases from Fedora. Where possible, they endeavour to stick to default Fedora release settings.

    The root partition is automatically re-sized when the OS is first booted. After re-sizing, it will automatically reboot. After rebooting the system will launch the 'initial-setup' utility. For graphical images this will occur on the display, for minimal images this will occur on the serial console. Failure to complete the initial-setup will prevent logging into the system.
